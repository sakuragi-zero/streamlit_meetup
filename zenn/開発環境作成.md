事前準備が完了していない場合は先に事前準備を完了してください。

### Clone Repositoryとコンテナ接続
リポジトリのクローンを行い新規で作業用のフォルダを作成します。
1. Clone Repositoryをクリックし以下のコマンドをコピー＆ペースト
```bash
https://github.com/sakuragi-zero/streamlit_meetup.git
```
![](https://storage.googleapis.com/zenn-user-upload/241b18276921-20260112.png =650x)

2. リポジトリのクローンを行うので既存のフォルダまたは新規でフォルダを作成します。
![](https://storage.googleapis.com/zenn-user-upload/b55b0795d152-20260112.png =700x)

3. クローンしたリポジトリを開く
![](https://storage.googleapis.com/zenn-user-upload/23265af86989-20260112.png =700x)

4. Devcontainer接続
開発用のコンテナ環境を使うように設定を行います。
※Docker Desktop が起動しているか確認してください。起動していない場合は、この後の手順がエラーになります。
command + shift + p でコマンドパレットを開きます。
```reopen in contain```と入力
画面上部にVSC-streamlit_meetupと表示されていれば完了です。
![](https://storage.googleapis.com/zenn-user-upload/6765eb430054-20260112.png =700x)
またcommand + jでターミナルを開きvscode -> /workspacesと表示されているかどうかでも確認することができます。
![](https://storage.googleapis.com/zenn-user-upload/a97fe7ccc51a-20260112.png =700x)

### 環境チェック
ワークフロー機能を使い環境が構築できているかチェックを行います。
チャット欄に```/check-env```と入力し送信します。
![](https://storage.googleapis.com/zenn-user-upload/7803da4aebe5-20260112.png =650x)

ライブラリが足りない場合はエージェントが教えてくれます。
手順に沿ってインストールを進めてください。

ワークフローが終了したら環境構築は完了です。
