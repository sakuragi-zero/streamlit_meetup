### ① 必要なもの一覧

* PC（Windows または Mac）
  * Linuxでも可能だと思われますが、動作未検証です。以下のセットアップ手順でもサポートしていません。自力でセットアップしていただく分には問題ありません。
* Google アカウント（Antigravity 用）
* Docker

## ② Antigravity のインストール

👉 **公式サイト**
[https://antigravity.google/](https://antigravity.google/)

* Antigravity のインストーラを公式サイトからダウンロードして、実行
* Antigravity を初回起動
* Antigravity のエージェントの動作モードを指定。エージェントが予想外の動作をしないように、Secure Mode を指定するのが推奨
* Google アカウントでサインイン
* ファイルを開いたり作成したりできるところまで確認する

## ③ Docker のインストール

### ■ Windows の方

👉 **Docker Desktop for Windows（公式）**
[https://docs.docker.com/desktop/install/windows-install/](https://docs.docker.com/desktop/install/windows-install/)

* Windows 10 / 11

⚠️ Windows で Docker を導入する方法は複数あります。そのうち WSL2 + Docker Desktop で導入する方法が環境に起因するトラブルが少ないです。WSL2 + Docker Desktop による導入の手順は、以下の参考記事を確認してください。

👉 参考記事（WSL2 + Docker Desktop）
[https://learn.microsoft.com/ja-jp/windows/wsl/tutorials/wsl-containers](https://learn.microsoft.com/ja-jp/windows/wsl/tutorials/wsl-containers)

---

### ■ Mac の方

👉 **Docker Desktop for Mac（公式）**
[https://docs.docker.com/desktop/install/mac-install/](https://docs.docker.com/desktop/install/mac-install/)

* Intel / Apple Silicon（M1/M2/M3）どちらも対応
* 基本的にはそのままインストールでOK

:::message
MacにDockerをインストールするとき、CLIのインストールにSystemでなくUserを選択した場合、`docker` コマンドにパスが通っておらず `docker --version` でエラーが出るかもしれません。
その場合、以下をログインシェルの設定ファイル（デフォルトでは `~/.zshrc` または `~/.zprofile` ）に追記してください。

```shell
export PATH="$HOME/.docker/bin:$PATH"
```

追記した後、シェルを再起動するか、以下のコマンドを実行してください。

```shell
source ~/.zshrc
```
（`~/.zprofile` に追記した場合は `source ~/.zprofile`）

:::

---

### ④ Docker が起動できるか確認

ターミナル（または PowerShell）で以下を実行：

```bash
docker --version
```

バージョンが表示されればOKです。

ツールのインストールはこれで全てです。お疲れ様でした！次のAntigravityの基本設定に進んでください。
